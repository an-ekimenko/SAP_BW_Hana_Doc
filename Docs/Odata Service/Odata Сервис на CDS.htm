Odata Сервис на CDS.

16.09.202216.09.2022 ~ Vyacheslav

Построение Odata сервисов с помощью CDS, без использования ABAP и Bex.

Odata Сервисы можно делать на CDS, помимо создания Odata Сервисов через ABAP и BEx Query.

Сначала можно изучить о CDS в целом в следующих статьях на русском:

Практика освоения ABAP CDS для непрограммистов. Часть 1  — в общем о CDS (что это такое).

Практика освоения ABAP CDS для непрограммистов. Часть 2 — как установить среду разработки для создания CDS (установка Eclipse, ADT, etc..).

Практика освоения ABAP CDS для непрограммистов. Часть 3 — основной пример, как создать простейший CDS ракурс.

Практика освоения ABAP CDS для непрограммистов. Часть 4 — рассказывается про ассоциации и как их использовать.

CDS Odata сервисы могут выдавать значения по указанным критериям, переданными как параметр внутри адреса. Кроме того CDS Odata сервисы могут передать Input parameter в код или в CV.

Аннотация @OData.publish: true позволяет сделать этот CDS как Odata сервис.

В транзакции /n/IWFND/MAINT_SERVICE можно добавить этот Odata сервис и протестировать его.

Для передачи Input параметра и получения нескольких значений по неуникальному ключу используется следующий синтаксис вызова сервиса: /sap/opu/odata/sap/ZI_CM_CALENDAR_DATE_CDS/ZI_CM_CALENDAR_DATE(P_CutDate=’20170217′,P_cutofftype=’EOD’)/Set.

Стоит обратить внимание, что указанный выше синтаксис по передаче параметров, транзакция для тестирования не предлагает в качестве шаблона. По кнопке EntitySets будет предложен вариант ZI_CM_CALENDAR_DATESet. В него параметр передаётся с синтаксисом: /sap/opu/odata/sap/ZI_CM_CALENDAR_DATE_CDS/ZI_CM_CALENDAR_DATESet(P_CutDate=’20170217′,CutDate=’20170217′). Такой вызов сервиса может вернуть только одно ключевое значение!  И если их несколько, то такой вызов будет выдавать ошибку. Для возможности возврата из сервиса нескольких значений по параметру нужно использовать синтаксис, который я показывал выше, т.е. использовать ZI_CM_CALENDAR_DATE вместо ZI_CM_CALENDAR_DATESet. Кроме того необходимо добавлять /Set в конце адреса с вызовом.

Реклама

Использование аннотации @EndUserText.label переда каждым полем позволяет отображать в сервисе лэйбл (описание) поля в сервисе. Т.е. например ввод такой аннотации перед кодом с валютой :

@EndUserText.label: ‘Валюта’
CURRENCY

отобразит в сервисе кроме технического имени поля в атрибуте «sap: label» русское описание поля.

Что бы посмотреть метаданные, необходимо вконце написать «$metadata», т.е. в данном примере …ZI_CM_CALENDAR_DATE/$metadata

Программа RUTDDLSACT помогает активировать CDS Ракурсы (источник DDL) в целевой системе при переносе.
